Step 3 - Retrieval Pipeline

- Semantic search model:
  - BAAI/bge-small-en-v1.5 embeddings in hybrid_retrieval.py.
- BM25 implementation:
  - rank-bm25 BM25Okapi over locally loaded chunk texts.
- Hybrid scoring formula:
  - score = semantic_weight * semantic_norm + bm25_weight * bm25_norm
  - semantic_norm, bm25_norm are min-max normalized channels.
  - source-adjusted score = score * source_multiplier(metadata)
- Weights used (submission run):
  - semantic_weight = 0.5
  - bm25_weight = 0.5
- Top-k value:
  - top_k = 6 (submission_20_query_eval.py).
- Reranking model:
  - cross-encoder/ms-marco-MiniLM-L-6-v2.
- Final retrieval pipeline:
  - Query -> Query Embedding -> Hybrid (Semantic + BM25) -> Cross-Encoder Rerank -> Generation


Step 4 - Faithfulness Evaluation Design

Faithfulness Evaluation Design
- Model used for judge:
  - Primary judge model: mistralai/Mistral-7B-Instruct-v0.2 (HF_EVAL_MODEL / HF_GENERATION_MODEL fallback chain).
  - If remote fails: local seq2seq LLM fallback model (LOCAL_FALLBACK_MODEL, default google/flan-t5-small).
- Claim extraction method:
  - Sentence splitting with regex: (?<=[.!?])\s+
  - Each sentence treated as a claim.
- Prompt used (remote judge):
  - System: Return only one label: SUPPORTED or NOT_SUPPORTED.
  - User prompt:
    - You are an expert evaluator.
    - Claim: <claim>
    - Context: <retrieved_context>
    - Is this claim supported by the context?
    - Answer only:
      - SUPPORTED
      - NOT_SUPPORTED
- Scoring formula:
  - Let N = number of claims.
  - Let S = number of claims labeled SUPPORTED.
  - Faithfulness score = S / N.
- Example output fields:
  - claim
  - label (SUPPORTED / NOT_SUPPORTED)
  - supported (boolean)
  - raw_response
  - backend (chat.completions / text_generation / local_llm_fallback / heuristic_fallback)


Step 5 - Relevance Evaluation Design

Relevance Evaluation Design
- Question generation model:
  - Primary: mistralai/Mistral-7B-Instruct-v0.2 (chat completions).
  - Fallback: local seq2seq LLM (google/flan-t5-small by default).
- Number of generated questions:
  - Target 3 questions.
- Embedding model for scoring:
  - BAAI/bge-small-en-v1.5.
- Similarity metric:
  - Cosine similarity between original query embedding and each generated-question embedding.
- Scoring formula:
  - For generated questions q1, q2, q3:
  - sim_i = cosine(Embed(query), Embed(q_i))
  - relevance_score = mean(sim_1, sim_2, sim_3)
